# 结构化视觉记忆组件（SVM）技术栈总结

## 项目定位

解决 LLM Agent 在长任务链和跨会话场景中的上下文遗忘问题，通过显式结构化记忆 + 触发召回机制，确保关键信息在需要时精准注入上下文前端。

---

## 整体架构：分层解耦

| 层级 | 职责 | 技术选型 | 理由 |
|-----|------|---------|------|
| **编排层** | 触发策略、LLM 调用、业务逻辑、与 OpenClaw 集成 | Python | 生态成熟，快速迭代，直接对接 OpenClaw 的 Python 框架 |
| **核心引擎** | 记忆存储、触发匹配、并发处理、召回排序 | Rust | 原生性能、无 GC 停顿、fearless concurrency、单二进制部署 |
| **通信层** | 跨语言服务调用、接口契约 | gRPC + protobuf | 强类型、低延迟、跨语言、版本兼容、进程隔离 |

---

## 核心引擎（Rust）技术栈

| 模块 | 技术选型 | 选型理由 |
|-----|---------|---------|
| 存储引擎 | SQLite（默认）/ Redis（缓存）/ Qdrant（向量扩展） | SQLite 零配置单机首选；Redis 加速热点查询；Qdrant 预留语义检索扩展 |
| 触发匹配 | Aho-Corasick 多模式匹配 | O(n) 复杂度，支持数千关键词同时匹配，优于正则性能 |
| 序列化 | serde + protobuf | 内部 Rust 结构用 serde，跨边界用 protobuf 契约 |
| 并发模型 | Tokio 异步运行时 | Rust 生态标准，无锁并发，高并发记忆查询不阻塞 |
| 部署形态 | 单二进制静态链接 | 跨平台复制即用，无动态库依赖，Docker 镜像 < 20MB |

---

## 编排层（Python）技术栈

| 模块 | 技术选型 | 选型理由 |
|-----|---------|---------|
| 框架集成 | OpenClaw 插件接口 | 原生对接，复用现有 Agent 生命周期和工具链 |
| gRPC 客户端 | grpcio + 生成代码 | Google 官方实现，与 Rust 服务端版本严格兼容 |
| 触发策略 | 规则引擎 + 轻量模型 | 规则保证确定性，轻量模型处理语义模糊匹配（如"那个项目"指代消解） |
| 上下文注入 | 模板引擎 | 控制记忆块在 System Prompt 中的位置、标记边界、防污染 |

---

## 通信与接口契约

| 维度 | 设计决策 |
|-----|---------|
| 协议 | gRPC over HTTP/2，保留 Unix Domain Socket 选项（单机部署降延迟） |
| 序列化 | protobuf 3，字段只增不减，向后兼容 |
| 服务发现 | 环境变量配置（默认 `localhost:50051`），K8s 场景支持 DNS 服务名 |
| 健康检查 | gRPC 标准 health check 协议，Rust 服务端内置，Python 客户端可探测 |

---

## 记忆数据模型

| 层级 | 结构 | 说明 |
|-----|------|------|
| 原子单元 | Memory Block | 键值对 + 元数据（创建时间、任务 ID、显式权重、过期策略） |
| 聚合单元 | Memory Slot | 同任务/同实体的 Block 集合，独立生命周期管理 |
| 召回单元 | Context Bundle | 按 token 预算截断后的有序 Block 列表，直接注入 LLM 上下文 |

---

## 关键设计原则

| 原则 | 实现 |
|-----|------|
| 先验 > 公式 | 触发策略优先用语义规则和用户显式标签，而非纯统计匹配 |
| 验证 > 聚类 | 召回的记忆块需通过时效性、任务关联度、用户权重三重验证，而非简单相似度排序 |
| 局部 Patch > 全局统计 | 每轮对话只召回当前任务相关的记忆槽，不加载全量历史 |
| 显式边界 > 隐式融合 | 记忆块用 `[Memory Context]` 标记包裹，明确区分背景资料与当前输入 |

---

## 部署拓扑（可选）

| 场景 | 部署方式 |
|-----|---------|
| 单机个人 Agent | Python + Rust 同机，gRPC over Unix Domain Socket |
| 多用户服务 | Python Agent 集群 + Rust Memory Service 集群，K8s 独立扩缩容 |
| 边缘-云混合 | Rust Memory 跑边缘设备（低资源），Python Agent 跑云端，网络 gRPC |

---

## 扩展预留

| 方向 | 预留接口 |
|-----|---------|
| 向量语义检索 | Qdrant 替代 SQLite 的模糊匹配，protobuf 接口不变 |
| 多租户隔离 | `tenant_id` 字段已加入契约，Rust 层实现逻辑隔离 |
| 记忆审计 | gRPC 拦截器 + 异步日志流，独立 Audit Service 消费 |
| 跨 Agent 共享 | Memory Service 天然多实例共享，只需统一地址配置 |

---

## 技术栈一句话总结

**Rust 筑基（存储+匹配+并发），Python 牵线（策略+集成+生态），gRPC 架桥（契约+通信+隔离），protobuf 立约（版本+兼容+扩展）。**
